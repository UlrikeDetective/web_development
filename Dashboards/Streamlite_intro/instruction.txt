Setup
pip install streamlit pandas plotly

# Run any app with:
streamlit run app.py

Your browser opens automatically at http://localhost:8501. Every time you save app.py, the page refreshes. That's the whole development loop.

🏗️ How Streamlit Actually Works
One important mental model before writing code:
Every time a user interacts with a widget, slider, button, or dropdown, Streamlit reruns your entire script from top to bottom with the new values.
This is different from traditional web frameworks.
There’s no manual event handling. You just write logic that uses widget values, and Streamlit handles the rest.

Your First App
Save this as app.py and run it:

import streamlit as st
import pandas as pd
import plotly.express as px

st.set_page_config(page_title="Sales Dashboard", layout="wide")

st.title("📊 Sales Dashboard")
st.markdown("Explore regional sales performance across product categories.")
st.set_page_config should always be the first Streamlit command. layout="wide" uses the full browser width instead of a narrow centred column.

Loading Data with Caching
Without caching, your data reloads every time a user touches a widget. With @st.cache_data, it loads once and stays in memory.

@st.cache_data
def load_data():
    df = pd.read_csv("sales.csv", parse_dates=["order_date"])
    return df

df = load_data()
Use @st.cache_data for data and DataFrames.

Use @st.cache_resource for heavier objects like database connections or ML models. Both persist across reruns until you change the underlying function.

Sidebar Filters
The sidebar is where filters live. It keeps your main canvas clean.

st.sidebar.header("🔍 Filters")

regions = st.sidebar.multiselect(
    "Select Region",
    options=df["region"].unique(),
    default=df["region"].unique()
)

date_range = st.sidebar.date_input(
    "Date Range",
    value=[df["order_date"].min(), df["order_date"].max()]
)

# Apply filters
filtered_df = df[
    (df["region"].isin(regions)) &
    (df["order_date"] >= pd.Timestamp(date_range[0])) &
    (df["order_date"] <= pd.Timestamp(date_range[1]))
]
Common widgets:

st.sidebar.selectbox : single choice dropdown
st.sidebar.multiselect : multiple choices
st.sidebar.slider : numerical range
st.sidebar.date_input : date or date range picker
st.sidebar.checkbox : boolean toggle

Layout with Columns
Columns let you place content side by side instead of stacked vertically.

col1, col2, col3 = st.columns(3)

col1.metric("Total Revenue", f"₹{filtered_df['revenue'].sum():,.0f}",
            delta="+12% vs last month")

col2.metric("Total Orders",  f"{len(filtered_df):,}")

col3.metric("Avg Order Value",
            f"₹{filtered_df['revenue'].mean():,.0f}")
st.metric is perfect for KPI cards. It shows a value with an optional delta indicator showing up or down change with colour coding.

For charts side by side:

left, right = st.columns(2)


with left:
    st.subheader("Revenue by Region")
    fig = px.bar(
        filtered_df.groupby("region")["revenue"].sum().reset_index(),
        x="region", y="revenue", color="region"
    )
    st.plotly_chart(fig, use_container_width=True)

with right:
    st.subheader("Orders Over Time")
    monthly = filtered_df.groupby(
        filtered_df["order_date"].dt.to_period("M")
    )["revenue"].sum().reset_index()
    monthly["order_date"] = monthly["order_date"].astype(str)
    fig2 = px.line(monthly, x="order_date", y="revenue")
    st.plotly_chart(fig2, use_container_width=True)

    Always pass use_container_width=True to charts. It makes them scale properly regardless of screen size.

🧩 Core UI Elements
You don’t need all of these but knowing they exist saves a lot of time:

Displaying data:

st.dataframe(df) : interactive sortable table
st.table(df) : static table
st.json(data) : formatted JSON viewer
st.code(snippet, language="python") : syntax-highlighted code block
Text and structure:

st.title, st.header, st.subheader : heading levels
st.markdown("**bold**, _italic_") : full Markdown support
st.divider() : horizontal rule between sections
st.expander("Click to expand") : collapsible section

User input:

st.text_input("Search") : text field

Feedback and status:

st.warning("No data for this filter") : yellow warning

st.spinner("Loading...") : loading indicator while something runs
🔄 Session State
Because Streamlit reruns on every interaction, regular Python variables reset. Session state lets values persist across reruns.

if "count" not in st.session_state:
    st.session_state.count = 0

if st.button("Click me"):
    st.session_state.count += 1

st.write(f"Button clicked {st.session_state.count} times")
Session state is essential for:

Tracking what a user has done across multiple interactions
Storing the result of an expensive computation after a button click
Building multi-step workflows like a form wizard
📄 Multi-Page Apps
Once your dashboard grows beyond one screen, split it into pages. Create this folder structure:

your_app/
├── app.py            ← entry point, shared sidebar
└── pages/
    ├── 1_Overview.py
    ├── 2_Regional.py
    └── 3_Products.py

Any .py file inside pages/ automatically appears as a navigation item in the sidebar. The number prefix controls the order. Each page is a normal Python script with its own st.title and content.

⚡ Forms: Batch Multiple Inputs
Without st.form, every widget interaction triggers a full rerun. For dashboards with many filters, this makes the app feel laggy. Group widgets in a form so the app only reruns when the user clicks Submit.

with st.form("filter_form"):
    region    = st.selectbox("Region", df["region"].unique())
    min_value = st.number_input("Minimum Order Value", value=0)
    submitted = st.form_submit_button("Apply Filters")

if submitted:
    result = df[(df["region"] == region) & (df["revenue"] >= min_value)]
    st.dataframe(result)

Deploying Your Dashboard
Streamlit Community Cloud (Free):

Push your code to a public GitHub repo
Go to share.streamlit.io
Connect the repo, pick the file, deploy
Add a requirements.txt at the root of your repo:

streamlit
pandas
plotly
That’s the entire deployment process.

Docker (For private or production apps):

FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
EXPOSE 8501
CMD ["streamlit", "run", "app.py", "--server.port=8501", "--server.address=0.0.0.0"]

Build, push to any cloud provider, done.

🎨 Theming
Create .streamlit/config.toml in your project root to customise colours without touching CSS:

[theme]
primaryColor        = "#017369"
backgroundColor     = "#38A6A5 "
secondaryBackgroundColor = "#BFB6AF"
textColor           = "#F2C9B3"
font                = "sans serif"

This gives you a dark theme with your brand’s primary colour in seconds.
